---
description: Generate and/or apply Alembic migrations for the Gateway DB
argument-hint: [--new "message"] | [--up]
---

Run Alembic against the running Postgres container.

Argument behavior:
- `--new "message"` → autogenerate a new migration: `docker compose run --rm app alembic revision --autogenerate -m "$MESSAGE"`. Then **show the generated file** to the user before applying — Alembic mis-detects index renames and `server_default` changes.
- `--up` → apply pending migrations: `docker compose run --rm app alembic upgrade head`.
- No arg or unrecognized arg → show current revision (`docker compose run --rm app alembic current`) and pending heads (`docker compose run --rm app alembic heads`).

Never apply an autogenerated migration without showing it to the user first.
